# Goals

- Prove the agent actually RECEIVES a fabric EventEnvelope end-to-end: forge write → CDC → fabric subscribe → agent's /fabric/subscribe SSE. Requires a dev IdP (a tiny JWKS the fabric gateway trusts via GATEWAY_JWKS_URL + a matching bearer the smoke mints) and a Keto seed step (write subscribe/view relation tuples for the subject/channel/envelope). The real-sibling stack + SSE bridge are already in place (p10-c005, p10-c004); this adds the missing auth layer.
- The dev IdP must be smoke-owned (nothing written into ../flint-realtime-fabric): either a tiny JWKS served from a static file in the compose stack (nginx or similar), or a well-known JWKS-compatible library the smoke uses to mint + serve tokens. Verify this is sufficient for fabric's OryIdentityVerifier (JWKS URL + Bearer + audience/issuer checks).
- The Keto seed: fabric's per-event view check requires a relation tuple (subject, view, envelope_id) — but envelope_id is random at publish time. Determine whether Keto supports a wildcard/subject-set grant (subject view * on namespace/object) that would cover ALL envelopes for a given subject, or whether we must seed the tuple AFTER publishing (before the subscriber drains). Grounding the exact Keto tuple model against frf-authz-keto before speccing is mandatory.
